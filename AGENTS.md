# AGENTS: AI 開発エージェント協調定義書

## 1. エージェント定義と役割分担
本プロジェクトの開発に関わるAIエージェント（Antigravityおよびサブエージェント）の役割分担を定義します。

| エージェント | 役割 | 担当スコープ |
| :--- | :--- | :--- |
| **Antigravity (メイン)** | 設計統括・Swift/C++メイン実装 | システム設計、ホスト・クライアントのソースコード実装、トラブルシューティング、ビルド検証。 |
| **Research (サブ)** | 技術調査・ドキュメント要約 | macOS/iOSプライベートAPIの仕様調査、Apple Developer Forum上の不具合情報の解析。 |
| **Self (サブ)** | コンテキスト分岐実行 | 大規模リファクタリングやパフォーマンスプロファイリングのテストコード記述など、独立したタスク実行。 |

---

## 2. 協調と情報同期ルール
- **コンテキスト共有**: 開発の状況、バグトラッキング、今後の計画は常に `PLAN.md`, `BUG.md`, `TASK.md` に最新化し、エージェント間で「これらのドキュメントを真っ先に読み込む」ことでコンテキストを同期する。
- **ドキュメントの整合性**: コード内のドキュメントコメント、および設計書の齟齬を発生させない。

---

## 3. Token Economy 原則（超重要ルール）
無駄な探索やトークン消費を避けるため、全てのエージェントは以下のルールを厳守すること。
1. **不要な全体調査の禁止**: ファイルを読む際は、必要最小限の範囲（行指定）で読み込む。全ファイルをロードしない。
2. **解決策は原則1択**: 複数のアプローチの冗長な比較は行わず、実証済みの「最小かつ最も安全な解決策」を1つだけ提示する。
3. **説明は最小限、実装優先**: 冗長で過剰に丁寧な説明は行わず、コードの差分や動作確認のコマンドの提示を最優先とする。
4. **差分変更の徹底**: ファイルを書き換える際は、全体の上書きを避け、必要な箇所のみを置換（`replace_file_content`）する。
